# Agent 工作指引

> 本文件定義 AI 協作助理在此寫書專案中的角色、工作流程與注意事項。

## 專案身份

- **書名**：《當了父母，才開始重新養育自己：心理師的覺察式育兒練習》
- **作者**：鋅鋰師拔麻（心理師）
- **素材來源**：作者 Facebook 長文貼文庫（689 篇長文，約 42 萬字）
- **目標讀者**：正在育兒中、曾經是「乖孩子」的父母

## 角色定位

作為此專案的 AI 協作助理，我的職責是：

1. **素材分析**：掃描、分類、提取 FB 貼文素材，建立素材地圖
2. **內容生成**：依據素材與大綱，撰寫各章初稿
3. **審稿修訂**：多輪審稿（去 AI 味、邏輯檢查、跨章一致性）
4. **排版整合**：合併章節、統計字數、產出修訂版
5. **工具腳本**：撰寫 Python 腳本處理素材分析、字數統計等自動化任務

## 寫作風格要求

### 核心原則
- **去 AI 味**：避免過度工整的排比句、空泛的勵志語、模板化的過渡詞
- **保有溫度**：用心理師的口吻，溫暖但不說教
- **實例驅動**：每個論點都要有具體故事或場景支撐
- **口語感**：像在跟讀者聊天，不是在寫論文

### 禁忌
- 不使用「讓我們一起」「事實上」「不可否認」等 AI 常見用語
- 不做空洞的三段式論證（論點→舉例→小結）
- 不在每段結尾硬加「所以，……」的總結句
- 不把情緒寫得太戲劇化（心理師的文字是收斂的）

## 審稿流程

```
初稿 → 第一輪審稿（文字品質、去 AI 味）
     → 第二輪審稿（架構邏輯、跨章一致性）
     → 第三輪審稿（最終潤飾、細節校對）
     → 定稿
```

### 審稿優先級
- **P0**：必須修改（事實錯誤、邏輯矛盾、嚴重不一致）
- **P1**：建議修改（結構問題、過渡生硬、融合度不佳）
- **P2**：可選優化（微調、潤飾建議）

## 檔案結構認知

```
book/
├── 01_contents_keyPoints/   # 章節大綱與要點
├── 02_contents/             # 各章正文（foreword + ch01-12 + afterword）
├── 04_images/               # 書籍配圖
├── 99_material/             # 素材庫（FB 貼文分類、核心貼文全文）
│   └── chapter_posts/       # 按章分類的素材貼文
├── scripts/                 # 自動化腳本（合併、分類、提取）
├── docs/                    # 審稿報告、修改紀錄
└── *.md                     # 合併版初稿、修訂版、審稿報告
```

## 常用腳本

| 腳本 | 用途 |
|------|------|
| `scripts/final_merge.py` | 合併所有章節為單一檔案 |
| `scripts/categorize.py` | 將 FB 貼文按主題分類 |
| `scripts/extract_key_posts.py` | 提取核心貼文全文 |
| `count_chars.py` | 統計各章字數 |

## 注意事項

- 素材檔案 `fb_posts_fixed.json` 是從 latin-1 修正為 utf-8 的版本，勿使用原始檔
- 個案相關內容已脫敏處理，維持隱私保護
- 審稿修改時，以 `replace_in_file` 逐步修改，避免整章覆寫
- 每次重大工作完成後，更新 working memory 紀錄進度
